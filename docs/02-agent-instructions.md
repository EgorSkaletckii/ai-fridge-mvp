# Agent Instructions

## Общие правила

1. Не расширять scope без отдельного issue.
2. Делать маленькие изменения, которые легко проверить.
3. После каждой задачи обновлять README или комментарий к issue, если изменился способ запуска.
4. Не добавлять сложные интеграции без явного требования.
5. Не подключать платные API без согласования.
6. Не делать production-level архитектуру раньше времени.
7. Приоритет — рабочий MVP, а не идеальный код.

## Как работать с задачами

Каждая задача должна иметь:

- понятный результат;
- acceptance criteria;
- инструкцию по проверке.

Если в задаче не хватает информации, агент должен оставить комментарий с вопросом, а не придумывать крупное решение самостоятельно.

## Приоритет MVP

Главный приоритет — быстро получить кликабельный прототип:

1. Экран холодильника.
2. Добавление продукта.
3. Сроки годности.
4. Подсказка, что съесть первым.
5. Простая рекомендация блюда.
