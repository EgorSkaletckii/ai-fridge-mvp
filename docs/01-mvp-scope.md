# AI Fridge — MVP Scope

## Цель MVP

Собрать первую рабочую версию приложения, которая показывает ценность идеи:

Пользователь может добавить продукты в цифровой холодильник, увидеть сроки годности и получить подсказку, что съесть или приготовить в первую очередь.

## Что входит в MVP

### 1. Главный экран

Экран "Мой холодильник".

На экране отображается список продуктов:

- название продукта;
- категория;
- количество;
- срок годности;
- статус продукта.

Примеры статусов:

- свежий;
- скоро испортится;
- просрочен.

### 2. Добавление продукта вручную

Пользователь может добавить продукт через форму.

Поля формы:

- название продукта;
- категория;
- количество;
- дата покупки;
- срок годности.

### 3. Сортировка по сроку годности

Приложение должно показывать продукты так, чтобы пользователь видел, что нужно съесть в первую очередь.

Логика:

1. Просроченные продукты — выше всего.
2. Продукты, которые испортятся в ближайшие 3 дня — далее.
3. Остальные продукты — ниже.

### 4. Простая рекомендация блюда

Пользователь может нажать кнопку "Что приготовить?"

Приложение должно показать простую рекомендацию на основе текущих продуктов.

Для MVP достаточно простой логики или моковой AI-рекомендации.

### 5. Пустое состояние

Если продуктов нет, пользователь видит понятный empty state:

- объяснение, что холодильник пока пустой;
- кнопку "Добавить продукт".

## Что НЕ входит в MVP

В первую версию не включаем:

1. Реальное распознавание чеков.
2. Интеграции с доставками еды.
3. Сложные AI-рецепты.
4. Диеты и платные планы питания.
5. Push-уведомления.
6. Сканирование штрихкодов.
7. Семейный доступ.
8. Админ-панель.
9. Мобильные публикации в App Store / Google Play.

## Рекомендуемый технический стек для MVP

Frontend:

- Next.js
- React
- TypeScript
- Tailwind CSS

Backend / Database:

- Supabase или PostgreSQL

Для первой итерации можно начать с моковых данных без базы.

## Критерий готовности MVP v0.1

MVP считается готовым, если пользователь может:

1. Открыть приложение.
2. Увидеть экран "Мой холодильник".
3. Добавить продукт.
4. Увидеть продукт в списке.
5. Понять, какие продукты скоро испортятся.
6. Получить простую рекомендацию, что приготовить.
